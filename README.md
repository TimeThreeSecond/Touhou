## 动作空间

**离散动作：288 种组合**

```
方向(9) × 冲刺(2) × 跳跃(2) × 左符卡(2) × 右符卡(2) × 射击(2) = 288
```

| 动作维度 | 取值 | 说明 |
|----------|------|------|
| 方向 | 0~8 | 0:静止, 1:上, 2:右上, 3:右, 4:右下, 5:下, 6:左下, 7:左, 8:左上 |
| 冲刺 | 0/1 | 0:释放, 1:按住 LeftShift |
| 跳跃 | 0/1 | 0:释放, 1:按住 Space |
| 左符卡 | 0/1 | 0:释放, 1:按住 Q |
| 右符卡 | 0/1 | 0:释放, 1:按住 E |
| 射击 | 0/1 | 0:释放, 1:按住鼠标左键 |

## 观察空间

**帧堆叠灰度图像：** `(4, 180, 320)`

- 输入：游戏画面截图
- 预处理：缩放至 320×180，转为灰度，归一化到 [0, 1]
- 帧堆叠：4 帧（用于捕捉弹幕运动信息）
- 最终观察形状：`(4, 180, 320)`

## 快速开始

### 1. 环境检查

```bash
cd D:\Touhou_script\script
python check_env.py
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 配置按键（如需要修改）

```bash
python key_config.py
```

- 选择 `2` 进入按键配置向导
- 默认已配置为实际游戏按键，一般无需修改

### 4. 检测游戏窗口

启动游戏后：

```bash
python key_config.py
```

- 选择 `3` 检测游戏窗口

### 5. 测试截图与状态检测

```bash
python utils.py
```

- 按 `S` 保存截图
- 按 `D` 显示检测到的状态（血量、体力、Boss 血量等）
- 按 `Q` 或 `Ctrl+C` 退出

### 6. 开始训练

```bash
python train.py
```

- 按 `Ctrl+C` 可随时暂停
- 模型自动保存到 `models/` 目录

### 7. 运行训练好的模型

```bash
python play.py --model models/best.pt
```

## 网络结构

```
输入: (4, 180, 320) 帧堆叠灰度图
  ↓
Conv2D(32, 8×8, stride=4) + ReLU
  ↓
Conv2D(64, 4×4, stride=2) + ReLU
  ↓
Conv2D(64, 3×3, stride=1) + ReLU
  ↓
展平 → 全连接(512) + ReLU → 全连接(256) + ReLU
  ↓
策略网络 → 输出 288 维动作概率
价值网络 → 输出 1 维状态价值
```

## 注意事项

1. **游戏窗口必须在前台**：截图和按键模拟都需要游戏窗口可见且在前台
2. **分辨率自适应**：UI 检测使用相对坐标，自动适应不同分辨率
3. **颜色阈值可能需要微调**：如果你的显示器或游戏画质设置不同，可能需要调整 `utils.py` 中 HSV 颜色范围
4. **首次运行建议先测试**：运行 `python utils.py` 确认状态检测准确后再开始训练
5. **存档备份**：训练过程中游戏会被自动控制，建议备份游戏存档

## 常见问题

**Q: 检测不到游戏窗口？**  
A: 确保游戏已启动且未最小化。在 `config.py` 中手动设置 `GAME_WINDOW["region"]`。

**Q: 截图全黑？**  
A: 游戏窗口必须在前台。某些全屏模式可能导致截图失败，尝试窗口化运行游戏。

**Q: 血量检测不准确？**  
A: 调整 `utils.py` 中 `detect_health()` 的 HSV 蓝色范围（`lower_blue` / `upper_blue`）。

**Q: 按键没有反应？**  
A: 运行 `python key_config.py` 选择 `4` 测试按键，确认游戏内按键配置与 `config.py` 一致。
