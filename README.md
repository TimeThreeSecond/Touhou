## 战斗画面 UI 检测

系统通过 **HSV 颜色检测** 实时提取以下游戏状态：

| UI 元素 | 位置 | 检测方式 | 用途 |
|---------|------|---------|------|
| 雪花血量 (0-6) | 左上角 | HSV 蓝色投影峰值计数 | 玩家生存状态 |
| 体力条 (0.0-1.0) | 雪花下方 | HSV 黄色列占比 | 冲刺资源管理 |
| Boss 血条 (0.0-1.0) | 右上角 | HSV 红色列占比 | 输出奖励计算 |
| Boss 阶段 (1-6) | 右侧 | 红色/白色文字存在性 + HP 突变推断 | 阶段突破奖励 |
| 符卡就绪 | 左侧 | READY 黄色 HSV 检测 | 技能使用判断 |

检测区域使用 **相对坐标**（百分比），自动适配不同分辨率。基准分辨率 **1600×900**。

## 奖励机制

| 奖励/惩罚 | 数值 | 触发条件 |
|-----------|------|---------|
| 存活奖励 | +0.05/帧 | 每帧存活 |
| 时间惩罚 | -0.005/帧 | 鼓励快速击杀 |
| 被击惩罚 | -5.0/雪花 | 被击中损失血量 |
| 回血奖励 | +2.0/雪花 | 血量恢复 |
| Boss 伤害奖励 | +2.0 × 伤害量 × 10 | 对 Boss 造成伤害 |
| 阶段突破奖励 | +15.0/阶段 | 击败 Boss 一条命 |
| 击败 Boss 奖励 | +100.0 | 第 6 阶段血量归零 |
| 射击奖励 | +0.02/帧 | 持续按住鼠标左键 |
| 符卡使用 | **-0.05** | 释放 Q/E 符卡（惩罚防止滥用） |
| 体力过低惩罚 | -0.1 | 黄条耗尽无法冲刺 |
| 冲刺消耗惩罚 | -0.01 | 使用冲刺消耗体力 |

> **注意**：`spell_use_reward` 已从 +0.5 改为 -0.05，防止 AI 无脑 spam 符卡。

## 观察空间（多模态）

环境返回 **`(visual, structured)`** 元组：

**视觉输入**：`(4, 180, 320)` 帧堆叠灰度图
- 输入：游戏画面截图
- 缩放至 320×180
- 转为灰度 → 归一化到 [0, 1]
- 4 帧堆叠（捕捉弹幕运动信息）

**结构化输入**：`(6,)` 数值向量
- `health/6` — 玩家血量归一化 0~1
- `stamina` — 体力条 0.0~1.0
- `boss_health` — Boss 血量 0.0~1.0
- `boss_stage/6` — 阶段归一化 0~1
- `spell_left` — 左符卡就绪 0/1
- `spell_right` — 右符卡就绪 0/1

结构化状态与 CNN 视觉特征在特征层拼接，共同输入 Policy/Value 网络。

## 动作空间

**离散动作：288 种组合**

```
方向(9) × 冲刺(2) × 跳跃(2) × 左符卡(2) × 右符卡(2) × 射击(2) = 288
```

| 维度 | 取值 | 说明 |
|------|------|------|
| 方向 | 0~8 | 0:静止, 1~8:8方向 |
| 冲刺 | 0/1 | 按住 LeftShift |
| 跳跃 | 0/1 | 按住 Space |
| 左符卡 | 0/1 | 按住 Q |
| 右符卡 | 0/1 | 按住 E |
| 射击 | 0/1 | 按住鼠标左键 |

## 网络结构（双分支）

```
视觉分支                     结构化分支
(4, 180, 320)                (6,)
    ↓                            ↓
Conv2D(32, 8×8, s=4) + ReLU   FloatTensor
    ↓                            ↓
Conv2D(64, 4×4, s=2) + ReLU   
    ↓                            
Conv2D(64, 3×3, s=1) + ReLU   
    ↓                            ↓
展平 → 特征向量(512)        结构化向量(6)
    └────────┬─────────────────┘
             ↓
       拼接 → (518,)
             ↓
    ┌────────┴────────┐
    ↓                 ↓
Policy(518→288)    Value(518→1)
Softmax            标量
```

- CNN 提取视觉特征，与 6 维结构化状态拼接
- 旧版纯视觉模型（无 structured 分支）因维度不匹配无法加载，会被自动跳过

## 训练详情

### 算法参数

| 参数 | 值 |
|------|-----|
| 算法 | PPO (Proximal Policy Optimization) |
| 学习率 | 3e-4 |
| n_steps | 2048（每次收集步数） |
| batch_size | 64 |
| γ (折扣因子) | 0.99 |
| GAE λ | 0.95 |
| clip_range (裁剪范围) | 0.2 |
| ent_coef (熵奖励) | 0.01 |
| vf_coef (价值损失权重) | 0.5 |
| max_grad_norm | 0.5 |
| 更新 epochs | 4 |

### 训练策略

| 策略 | 值 | 说明 |
|------|-----|------|
| max_episode_steps | 10,000 | Gymnasium 硬上限 |
| max_length (AFK Guard) | 5,400 (3min@30fps) | 超时后停止输入等待死亡 |
| 动态 reset | 循环检测 health>0 | 防止黑屏画面导致无效 episode |
| 训练时长 | 1h / 2h / 8h / 12h | 启动时交互式选择 |

### 模型保存策略

| 文件 | 保存条件 |
|------|---------|
| `models/latest.pt` | 每 `save_freq` 步覆盖保存 |
| `models/best.pt` | 平均奖励创新高时保存 |
| `models/model_step_N.pt` | 每 `save_freq` 步保存为时间点快照 |
| `models/final.pt` | 训练中断/完成时保存 |

## 快速开始

### 1. 环境检查

```bash
cd script
python check_env.py
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

核心依赖：torch, numpy, opencv-python, mss, gymnasium, pydirectinput, pywin32

### 3. 配置按键（可选）

```bash
python key_config.py
```

- 选项 `1` — 查看当前按键配置
- 选项 `2` — 进入按键配置向导
- 默认配置已与实际游戏按键一致，一般无需修改

### 4. 检测游戏窗口

启动游戏后：

```bash
python key_config.py
```

- 选项 `3` — 自动检测游戏窗口并显示截图区域坐标
- 使用 **pywin32**（主） + **ctypes**（备选）双路径检测
- 通过进程路径验证（`QueryFullProcessImageNameW`）排除误匹配

### 5. 测试截图与状态检测

```bash
python utils.py
```

- 按 `S` — 保存当前截图
- 按 `D` — 显示检测到的 UI 状态（血量、体力、Boss 血量等）
- 按 `Q` 或 `Ctrl+C` — 退出

> **建议**：在开始训练前先用此工具确认状态检测是否准确。

### 6. 开始训练

```bash
python train.py
```

启动后选择训练时长：

| 选项 | 时长 | 步数 | 约计 |
|------|------|------|------|
| 1 | ~1 小时 | 35,000 | 快速测试 |
| 2 | ~2 小时 | 70,000 | 短训 |
| 3 | ~8 小时 | 275,000 | 标准（默认） |
| 4 | ~12 小时 | 410,000 | 完整 |
| 5 | 自定义 | 手动输入 | — |
| 6 | 配置默认 | 1,000,000 | ~30 小时 |

- 自动加载已有模型（`models/latest.pt`），旧版模型自动跳过
- 按 `Ctrl+C` 可随时暂停，模型自动保存到 `models/` 目录

### 7. 运行训练好的模型

```bash
python play.py                     # 使用最佳模型
python play.py --model models/latest.pt  # 指定模型
python play.py --no-render          # 不显示画面（纯后台运行）
python play.py --fps 15            # 降低帧率
```

## 注意事项

1. **游戏窗口必须在前台**：截图（mss）和按键模拟（pydirectinput）都要求窗口可见且在前台
2. **分辨率自适应**：UI 检测使用相对坐标（百分比），自动适配不同分辨率（基准 1600×900）
3. **颜色阈值可能需要微调**：因显示器或游戏画质设置不同，可能需要调整 `utils.py` 中各检测方法的 HSV 颜色范围
4. **首次运行建议先测试**：运行 `python utils.py` 确认状态检测准确后再开始训练
5. **存档备份**：训练过程中游戏会被自动控制，建议备份游戏存档
6. **GPU 推荐**：CNN 推理在 CUDA 上运行；CPU 模式训练速度会大幅下降
7. **速度优化**：已修复 `pydirectinput.PAUSE` 导致的 500~800ms 每步延迟，当前每步约 95ms（~10 FPS）

## 常见问题

**Q: 检测不到游戏窗口？**
A: 确保游戏已启动且未最小化。运行 `python key_config.py` 选 `3` 检测。如果仍未找到，运行同一工具的 `3` 号选项会列出所有可见窗口标题供排查。或在 `config.py` 中手动设置 `GAME_WINDOW["region"]`。

**Q: 截图全黑？**
A: 游戏窗口必须在前台。某些全屏模式可能导致截图失败，尝试窗口化运行游戏。

**Q: 血量检测不准确？**
A: 调整 `utils.py` 中 `detect_health()` 的 HSV 蓝色范围（`lower_blue` / `upper_blue`）。

**Q: 按键没有反应？**
A: 运行 `python key_config.py` 选择 `4` 测试按键。确认游戏内按键配置与 `config.py` 一致。某些游戏需要以管理员身份运行 Python 脚本才能模拟输入。

**Q: 训练时不收敛？**
A: 检查奖励数值是否合理（`python utils.py` 按 D 查看检测状态）。确认 Boss 血条颜色范围正确。尝试降低学习率或增加训练步数。

## 已知限制

- **环境检测完全基于视觉**：不读取游戏内存，所有状态通过 HSV 颜色检测从画面提取，受画面质量影响
- **无 DLC 适配**：DLC 内容（Rose Idol, Immortal Phoenix）的 UI 布局可能不同，需调整检测区域
- **动作组合有限**：288 种离散动作无法覆盖所有可能的输入组合（例如同时按两个方向键）
- **Boss 阶段通过血量突变推断**：非直接读取游戏数据，在高难度下多阶段连续突破时可能漏检
- **训练收敛性待验证**：当前奖励函数和超参数基于理论设计，需实际训练数据验证
- **旧模型不兼容**：结构化状态（方案A）之前的 `.pt` 模型文件无法加载
